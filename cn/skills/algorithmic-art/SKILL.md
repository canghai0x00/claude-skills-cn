算法哲学是通过代码表达的计算美学运动。输出.md 文件（哲学）、.html 文件（交互式查看器）和.js 文件（生成算法）。

这分为两个步骤：

1. 算法哲学创建（.md 文件）
2. 通过创建 p5.js 生成艺术来表达（.html + .js 文件）

首先，执行这个任务：

## 算法哲学创建

首先，创建一个算法哲学（不是静态图像或模板），它将通过以下方式解释：

- 计算过程、涌现行为、数学美
- 种子随机性、噪声场、有机系统
- 粒子、流、场、力
- 参数变化和受控混沌

### 关键理解

- 接收内容：用户提供的一些微妙输入或指示作为考虑因素，但仅作为基础；它不应限制创造自由。
- 创建内容：一种算法哲学/生成美学运动。
- 接下来发生的事：同一个版本接收这个哲学并通过代码表达它 - 创建 90%是算法生成，10%是基本参数的 p5.js 草图。

考虑这种方法：

- 为一种生成艺术运动写一份宣言
- 下一阶段涉及编写使其栩栩如生的算法

该哲学必须强调：算法表达。涌现行为。计算美学。种子变化。

### 如何生成算法哲学

**命名运动**（1-2 个词）："有机湍流"/"量子和声"/"涌现静态"

**阐述哲学**（4-6 段 - 简洁但完整）：

要捕捉算法本质，表达这种哲学如何通过以下方式体现：

- 计算过程和数学关系？
- 噪声函数和随机性模式？
- 粒子行为和场动态？
- 时间演变和系统状态？
- 参数变化和涌现复杂性？

**关键指南：**

- **避免冗余**：每个算法方面应该只提及一次。除非添加新的深度，否则避免重复关于噪声理论、粒子动力学或数学原理的概念。
- **反复强调工艺**：该哲学必须多次强调最终算法应该看起来像是花了无数小时开发，经过精心改进，来自该领域绝对顶尖的人才。这种框架是必不可少的 - 重复使用"精心打造的算法"、"深厚计算专业知识的产物"、"精心优化"、"大师级实现"等短语。
- **留出创造空间**：对算法方向要具体，但要足够简洁，以便下一个 Claude 有空间在极高水平的工艺基础上做出解释性实现选择。

该哲学必须指导下一个版本通过算法表达想法，而不是通过静态图像。美存在于过程中，而不是最终的帧中。

### 哲学示例

**"有机湍流"**
哲学：被自然法则约束的混沌，从无序中涌现的秩序。
算法表达：由分层 Perlin 噪声驱动的流场。数千个遵循矢量力的粒子，它们的轨迹积累成有机密度图。多个噪声八度创建湍流区域和平静区域。速度和密度产生颜色 - 快速粒子燃烧明亮，慢速粒子渐隐为阴影。算法运行直到达到平衡 - 一种精心调整的平衡，其中每个参数都是通过计算美学大师无数次迭代精炼的。

**"量子和声"**
哲学：表现出波状干涉图案的离散实体。
算法表达：在网格上初始化的粒子，每个粒子携带一个通过正弦波演变的相位值。当粒子靠近时，它们的相位互相干涉 - 建设性干涉创造明亮节点，破坏性干涉创造空隙。简单的谐波运动产生复杂的涌现曼陀罗。精心校准频率的结果，其中每个比率都经过精心选择，以产生共振美。

**"递归耳语"**
哲学：跨尺度的自相似性，有限空间中的无限深度。
算法表达：递归细分的分支结构。每个分支略微随机化但受黄金比例约束。L 系统或递归细分生成既数学又有机的树状形式。微妙的噪声扰动打破完美对称。线条权重随每个递归级别减小。每个分支角度都是深入数学探索的产物。

**"场动力学"**
哲学：通过其对物质的影响使无形力量变得可见。
算法表达：由数学函数或噪声构建的矢量场。粒子在边缘生成，沿场线流动，在达到平衡或边界时消亡。多个场可以吸引、排斥或旋转粒子。可视化只显示痕迹 - 无形力量的幽灵般证据。通过力平衡精心编排的计算舞蹈。

**"随机结晶"**
哲学：结晶为有序结构的随机过程。
算法表达：随机圆形填充或 Voronoi 镶嵌。从随机点开始，让它们通过松弛算法演变。细胞相互推开直到平衡。颜色基于细胞大小、相邻数量或与中心的距离。涌现的有机镶嵌感觉既随机又不可避免。每个种子产生独特的晶体美 - 大师级生成算法的标志。

_这些是浓缩的例子。实际的算法哲学应该是 4-6 个实质性段落。_

### 基本原则

- **算法哲学**：创建一个通过代码表达的计算世界观
- **过程重于产品**：始终强调美从算法执行中涌现 - 每次运行都是独特的
- **参数表达**：思想通过数学关系、力量、行为交流 - 而不是静态构图
- **艺术自由**：下一个 Claude 以算法方式解释哲学 - 提供创意实现空间
- **纯生成艺术**：这是关于制作活的算法，而不是带随机性的静态图像
- **专家工艺**：反复强调最终算法必须感觉精心制作，通过无数次迭代精炼，是计算美学领域绝对顶尖专家的深度专业知识的产物

**算法哲学应该是 4-6 段长。** 用将预期愿景融合在一起的诗意计算哲学填充它。避免重复相同的要点。将此算法哲学输出为.md 文件。

**基本原则**：

### 技术要求

**种子随机性（Art Blocks 模式）**：

```javascript
// 始终使用种子以实现可重现性
let seed = 12345; // 或来自用户输入的哈希
randomSeed(seed);
noiseSeed(seed);
```

**参数结构 - 遵循哲学**：

要建立自然从算法哲学中涌现的参数，考虑："这个系统的哪些品质可以调整？"

```javascript
let params = {
  seed: 12345, // 始终包含种子以实现可重现性
  // 颜色
  // 添加控制你的算法的参数：
  // - 数量（有多少？）
  // - 尺度（多大？多快？）
  // - 概率（多可能？）
  // - 比率（什么比例？）
  // - 角度（什么方向？）
  // - 阈值（行为何时改变？）
};
```

**设计有效参数，专注于系统需要可调性的属性，而不是以"模式类型"思考。**

**核心算法 - 表达哲学**：

**关键**：算法哲学应该决定构建什么。

要通过代码表达哲学，避免思考"应该使用哪种模式？"而是思考"如何通过代码表达这种哲学？"

如果哲学关于**有机涌现**，考虑使用：

- 随时间积累或成长的元素
- 受自然规则约束的随机过程
- 反馈循环和互动

如果哲学关于**数学美**，考虑使用：

- 几何关系和比率
- 三角函数和和声
- 精确计算创造意外模式

如果哲学关于**受控混沌**，考虑使用：

- 在严格边界内的随机变化
- 分岔和相变
- 从无序中涌现的秩序

**算法源自哲学，而不是选项菜单。**

为指导实现，让概念本质告知创新和原创选择。构建能表达这个特定请求视野的东西。

**画布设置**：标准 p5.js 结构：

```javascript
function setup() {
  createCanvas(1200, 1200);
  // 初始化你的系统
}

function draw() {
  // 你的生成算法
  // 可以是静态的（noLoop）或动画的
}
```

### 工艺要求

**关键**：要达到精通，创建感觉像是由大师级生成艺术家通过无数次迭代产生的算法。仔细调整每个参数。确保每个模式都有目的地涌现。这不是随机噪声 - 这是通过深厚专业知识精炼的受控混沌。

- **平衡**：复杂而无视觉噪声，有序而无呆板
- **色彩和谐**：经过深思熟虑的调色板，而非随机 RGB 值
- **构图**：即使在随机性中，也保持视觉层次和流动
- **性能**：流畅执行，如果是动画则为实时优化
- **可重现性**：相同种子始终产生相同输出

### 输出格式

输出：

1. **算法哲学** - 作为 markdown 或文本解释生成美学
2. **单一 HTML 工件** - 基于`templates/viewer.html`构建的自包含交互式生成艺术（见步骤 0 和下一节）

HTML 工件包含所有内容：p5.js（从 CDN）、算法、参数控制和 UI - 所有内容都在一个文件中，可以在 claude.ai 工件或任何浏览器中立即工作。从模板文件开始，而不是从头开始。

## 交互式工件创建

**提醒：应该已经阅读了`templates/viewer.html`（见步骤 0）。使用该文件作为起点。**

要允许探索生成艺术，创建一个单一的、自包含的 HTML 工件。确保此工件可以立即在 claude.ai 或任何浏览器中工作 - 无需设置。内联嵌入所有内容。

### 关键：什么是固定与可变的

`templates/viewer.html`文件是基础。它包含所需的确切结构和样式。

**固定（始终按原样包含）：**

- 布局结构（页眉、侧边栏、主画布区域）
- Anthropic 品牌（UI 颜色、字体、渐变）
- 侧边栏中的种子部分：
  - 种子显示
  - 上一个/下一个按钮
  - 随机按钮
  - 跳转到种子输入 + 执行按钮
- 侧边栏中的操作部分：
  - 重新生成按钮
  - 重置按钮

**可变（为每个艺术品自定义）：**

- 整个 p5.js 算法（setup/draw/类）
- 参数对象（定义艺术需要什么）
- 侧边栏中的参数部分：
  - 参数控件数量
  - 参数名称
  - 滑块的最小/最大/步长值
  - 控件类型（滑块、输入等）
- 颜色部分（可选）：
  - 某些艺术需要颜色选择器
  - 某些艺术可能使用固定颜色
  - 某些艺术可能是单色的（不需要颜色控件）
  - 根据艺术需求决定

**每个艺术品应该有独特的参数和算法！** 固定部分提供一致的用户体验 - 其他一切表达独特的视野。

### 必需功能

**1. 参数控件**

- 数值参数的滑块（粒子数量、噪声尺度、速度等）
- 调色板颜色的颜色选择器
- 参数更改时实时更新
- 恢复默认值的重置按钮

**2. 种子导航**

- 显示当前种子号
- "上一个"和"下一个"按钮循环浏览种子
- "随机"按钮获取随机种子
- 跳转到特定种子的输入字段
- 根据请求生成 100 个变体（种子 1-100）

**3. 单一工件结构**

```html
<!DOCTYPE html>
<html>
  <head>
    <!-- 从CDN加载p5.js - 始终可用 -->
    <script src="https://cdnjs.cloudflare.com/ajax/libs/p5.js/1.7.0/p5.min.js"></script>
    <style>
      /* 所有样式内联 - 简洁、最小化 */
      /* 画布在上，控制在下 */
    </style>
  </head>
  <body>
    <div id="canvas-container"></div>
    <div id="controls">
      <!-- 所有参数控件 -->
    </div>
    <script>
      // 所有p5.js代码内联于此
      // 参数对象、类、函数
      // setup()和draw()
      // UI处理程序
      // 一切自包含
    </script>
  </body>
</html>
```

**关键**：这是一个单一工件。没有外部文件，没有导入（除了 p5.js CDN）。一切内联。

**4. 实现细节 - 构建侧边栏**

侧边栏结构：

**1. 种子（固定）** - 始终按原样包含：

- 种子显示
- 上一个/下一个/随机/跳转按钮

**2. 参数（可变）** - 为艺术创建控件：

```html
<div class="control-group">
  <label>参数名称</label>
  <input
    type="range"
    id="param"
    min="..."
    max="..."
    step="..."
    value="..."
    oninput="updateParam('param', this.value)"
  />
  <span class="value-display" id="param-value">...</span>
</div>
```

添加与参数数量相同的 control-group div。

**3. 颜色（可选/可变）** - 如果艺术需要可调整颜色则包含：

- 如果用户应控制调色板，添加颜色选择器
- 如果艺术使用固定颜色，跳过此部分
- 如果艺术是单色的，则跳过

**4. 操作（固定）** - 始终按原样包含：

- 重新生成按钮
- 重置按钮
- 下载 PNG 按钮

**要求**：

- 种子控件必须工作（上一个/下一个/随机/跳转/显示）
- 所有参数必须有 UI 控件
- 重新生成、重置、下载按钮必须工作
- 保留 Anthropic 品牌（UI 样式，不是艺术颜色）

### 使用工件

HTML 工件立即工作：

1. **在 claude.ai 中**：显示为交互式工件 - 立即运行
2. **作为文件**：保存并在任何浏览器中打开 - 无需服务器
3. **分享**：发送 HTML 文件 - 它是完全自包含的

## 变体与探索

工件默认包含种子导航（上一个/下一个/随机按钮），允许用户在不创建多个文件的情况下探索变体。如果用户想突出显示特定变体：

- 包含种子预设（"变体 1：种子 42"、"变体 2：种子 127"等按钮）
- 添加"画廊模式"，并排显示多个种子的缩略图
- 全部在同一个单一工件内

这就像从同一个版画制作一系列印刷品 - 算法一致，但每个种子揭示其潜力的不同方面。交互性意味着用户可以通过探索种子空间发现自己喜欢的变体。

## 创作过程

**用户请求** → **算法哲学** → **实现**

每个请求都是独特的。该过程包括：

1. **解释用户意图** - 寻求什么美学？
2. **创建算法哲学**（4-6 段）描述计算方法
3. **在代码中实现它** - 构建表达此哲学的算法
4. **设计适当的参数** - 什么应该可调？
5. **构建匹配的 UI 控件** - 这些参数的滑块/输入

**不变的部分**：

- Anthropic 品牌（颜色、字体、布局）
- 种子导航（始终存在）
- 自包含 HTML 工件

**所有其他可变的部分**：

- 算法本身
- 参数
- UI 控件
- 视觉结果

为获得最佳结果，相信创造力并让哲学指导实现。
